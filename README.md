## v0.4:
Программа соотвествует MVP согласно тех.заданию, а именно:
* Полноэкранное окно со списком приложений, приложения запускаются по кнопке;
* Чтение конфигурации, модели данных, валидация
* Управление жизненным циклом приложений
* Логирование в консоль или в файл
* Обработка ошибок
---
### Общее
В релизе есть всё необходимое для тестирования: .bat для регистрации лаунчера в роли shell, сам лаунчер, runtime под него (чтобы не устанавливать jre самостоятельно). \
>[!WARNING]
>Лаунчер тестировался только в роли Windows Shell. Стабильная работа в роли обычного приложения поверх explorer не гарантируется
### Конфигурация
В лаунчер можно передать путь к конфигурации через --config (пример: launcher.exe --config=D:\conf.json). Выбор конфигурации происходит так: <br/><br/>
**параметр --config -> config.json в папке с launcher.exe -> предзаписанный стандартный конфиг, лежащий в .jar.** <br/><br/>
Пример конфига можно посмотреть здесь (config.json) или в релизе. Рекомендуется держать в секции apps "unreg" (пример есть в config.json), чтобы было проще вернуть стандартный shell (explorer.exe).\
MVP не предполагает наличие admin-меню, поэтому сочетание клавиш завершает работу лаунчера\
### Логирование:
Логировать можно в файл ("file") или в консоль ("console"). Если значение в конфиге указано неверно, по умолчанию лог идёт в консоль. Смысла в консоль логировать нет потому, что текущая сборка её не содержит.
